# 房产图片上传功能 - 需求与实施方案

## 确认版本 v1.0

### 1. 功能需求

| 需求项 | 描述 | 状态 |
|--------|------|------|
| 图片上传 | 点击/拖拽上传 | 待开发 |
| 数量限制 | 最多10张/房产 | 待开发 |
| 尺寸要求 | 最小300x300px | 待开发 |
| 图片压缩 | 过大自动压缩(≤1920px) | 待开发 |
| 缩略图 | 300x300px | 待开发 |
| 封面标识 | 第一张为封面 | 待开发 |
| 图片删除 | 删除已上传图片 | 待开发 |
| 图片预览 | 灯箱放大查看 | 待开发 |
| 数据迁移 | 现有房产数据迁移 | 待开发 |

### 2. 技术确认

| 项目 | 决策 |
|------|------|
| 存储方式 | 本地存储 (uploads/ 目录) | ✅ |
| 水印 | 不需要 | ✅ |
| 数据迁移 | 需要迁移现有数据 | ✅ |

### 3. 图片处理参数

| 参数 | 值 |
|------|------|
| 最小尺寸 | 300 x 300 px |
| 最大文件 | 10 MB |
| 支持格式 | JPG, PNG, GIF, WebP |
| 压缩质量 | 85% |
| 压缩阈值 | 1920 px |
| 缩略图尺寸 | 300 x 300 px |

### 4. API 设计

| 端点 | 方法 | 说明 |
|------|------|------|
| `/upload/image` | POST | 上传图片，返回原图/缩略图路径 |
| `/upload/delete` | POST | 删除图片 |

### 5. 数据模型

```python
# Property 模型新增字段
images = db.Column(db.Text, default='[]')  # JSON字符串
```

### 6. 文件结构

```
uploads/
├── {uuid}.jpg          # 原图
└── thumbs/
    └── thumb_{uuid}.jpg  # 缩略图
```

### 7. 数据库迁移

ALTER TABLE property ADD COLUMN images TEXT DEFAULT '[]';
