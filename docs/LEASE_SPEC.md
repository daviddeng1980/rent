### 4. 数据模型

```python
# Lease 模型状态说明
# status 字段值：
#   - 'pending': 待生效（房产ID和租客ID未同时匹配）
#   - 'active': 生效中（房产ID和租客ID都已匹配）
#   - 'renewed': 已续签
#   - 'terminated': 已终止
#   - 'expired': 已到期

# Lease 模型新增字段
contract_files = db.Column(db.Text, default='[]')  # JSON数组，存储文件信息
original_lease_id = db.Column(db.Integer, ForeignKey('lease.id'), nullable=True)  # 续签的原租约ID
renewed_lease_id = db.Column(db.Integer, ForeignKey('lease.id'), nullable=True)  # 续签生成的新租约ID

# 新增 LeaseChange 模型
class LeaseChange(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    lease_id = db.Column(db.Integer, db.ForeignKey('lease.id'), nullable=False)
    change_type = db.Column(db.String(20), nullable=False)  # 'amendment'/'termination'
    change_data = db.Column(db.Text)  # JSON存储变更详情（字段变更前后对比）
    old_values = db.Column(db.Text)  # 变更前的值
    new_values = db.Column(db.Text)  # 变更后的值
    effective_date = db.Column(db.Date)  # 变更生效日期
    reason = db.Column(db.String(200))
    operator = db.Column(db.String(50))  # 操作人
    create_time = db.Column(db.DateTime, default=db.func.now())

# Payment 模型状态
# status 字段值：
#   - 'pending': 未到期
#   - 'overdue': 逾期（已到期但未缴纳）
#   - 'paid': 已缴纳

# 终止原因枚举
TERMINATION_REASONS = ['到期终止', '提前终止', '违约终止']
```

### 5. API 设计

| 端点 | 方法 | 说明 |
|------|------|------|
| `/leases` | GET | 获取租约列表（包含状态筛选） |
| `/leases` | POST | 创建租约（初始为待生效状态） |

| `/leases/<id>/contracts` | GET | 获取合同文件列表 |

### 6. 租约生效流程

1. 创建租约时：
   - 租约初始状态为 "待生效"
   - 房产ID和租客ID必须都已选择
   - 可以先保存草稿，之后再确认生效
2. 确认生效：
   - 用户点击"确认生效"按钮
   - 系统验证房产和租客都已匹配
   - 状态变更为 "生效中"
   - 自动生成完整的付款计划
   - 房产状态更新为"出租中"

**注意**：未生效的租约不会生成付款记录。

### 7. 续签流程

1. 用户选择现有租约，点击"续签"
2. 系统预填充原租约的房产、租客信息
3. 新租约初始状态为"待生效"
4. 用户填写新租期的开始/结束日期，可调整租金
5. 提交后：
   - 原租约状态改为 "已续签"
   - 创建新租约，关联 original_lease_id
   - 原租约的付款记录全部标记为已完成
   - 新租约需要确认生效后才生成付款计划

### 8. 变更流程

1. 用户选择生效中的租约，点击"变更"
2. 可变更字段：租金、租金日、押金、备注
3. 选择生效方式：
   - 立即生效：从变更日起执行新租金
   - 下期生效：从下一个付款周期开始执行
4. 提交后：
   - 记录变更到 lease_changes 表（包含变更前后对比）
   - 如果是租金变更且选择立即生效，需要计算差价并生成补收/退还记录

### 9. 终止流程

1. 用户选择生效中的租约，点击"终止"
2. 选择终止原因（到期终止/提前终止/违约终止）
3. 填写终止日期和备注
4. 提交后：
   - 租约状态改为 "已终止"
   - 终止日期之后未付的款项标记为 "已取消"
   - 已逾期款项保留，追缴处理
   - 押金处理：记录押金退还信息
   - 房产状态更新为空置

### 10. 租金状态分类

| 状态 | 条件 | 颜色标识 |
|------|------|----------|
| 未到期 | 当前日期 < 到期日期 | 蓝色 |
| 逾期 | 当前日期 >= 到期日期 且 status = 'pending' | 红色 |
| 已缴纳 | 已支付 | 绿色 |

### 11. 租金到期提醒

- 付款计划生成时，系统计算每期到期日
- 距到期日30天内自动标记为"即将到期"
- 在仪表盘显示提醒列表
- 提醒列表按到期日期排序

### 13. UI 设计

**租约列表页**
- 状态筛选：全部/待生效/进行中/已续签/已终止
- 快速操作：查看/确认生效/续签/变更/终止
- 状态标签颜色：待生效(橙色)/进行中(绿色)/已续签(蓝色)/已终止(灰色)

**租约详情页**
- 基本信息卡片（显示生效状态）
- 付款计划列表（按状态分类显示）
- 变更历史时间线
- 合同文件列表（支持预览/下载）
- 操作按钮：续签/变更/终止

**创建租约弹窗**
- 房产选择（可选已空置房产）
- 租客选择（可选）
- 基础租金信息
- 保存为草稿（待生效）/确认生效 两种按钮

**续签弹窗**
- 预填充原租约信息（房产、租客）
- 新租约初始为"待生效"状态
- 新租期日期选择
- 新租金设置（默认继承原租金）
- 押金设置

**变更弹窗**
- 变更字段选择
- 原值 → 新值 对比显示
- 生效方式选择（立即/下期）
- 变更原因填写

**终止弹窗**
- 终止原因单选
- 终止日期选择
- 押金处理说明
- 备注

**仪表盘租金提醒**
- 显示未来30天内即将到期的租金
- 每条显示：房产名称、租客姓名、到期日期、金额
- 逾期租金高亮显示
